开发工具类（减少造轮子）

Java自带

CollectionsUtils

  • 用于简化集合（Collection、List、Map 等）的操作。它封装了多种集合操作的静态方法，可以安全处理空集合、实现集合的过滤、转换、合并等逻辑，避免开发者重复编写底层代码。

核心功能与用法

  • ​（1）判断集合是否为空

    import org.apache.commons.collections4.CollectionUtils;

List<String> list = new ArrayList<>();
boolean isEmpty = CollectionUtils.isEmpty(list); // true

List<String> list2 = Arrays.asList("a", "b");
boolean isNotEmpty = CollectionUtils.isNotEmpty(list2); // true

  • ​​（2）集合的交集、并集、差集

    List<String> listA = Arrays.asList("a", "b", "c");
List<String> listB = Arrays.asList("b", "c", "d");

// 交集：两个集合都包含的元素
Collection<String> intersection = 
    CollectionUtils.intersection(listA, listB); // ["b", "c"]

// 并集：合并两个集合的所有元素（去重）
Collection<String> union = 
    CollectionUtils.union(listA, listB); // ["a", "b", "c", "d"]

// 差集：A 中有但 B 中没有的元素
Collection<String> subtract = 
    CollectionUtils.subtract(listA, listB); // ["a"]

  • ​**（3）过滤集合元素 **

    List<Integer> numbers = Arrays.asList(1, 2, 3, 4, 5);

// 过滤出偶数
CollectionUtils.filter(numbers, n -> n % 2 == 0); 
// numbers 变为 [2, 4]（直接修改原集合！）

  • （4）集合转换

    List<String> strings = Arrays.asList("1", "2", "3");

// 将字符串转换为整数
Collection<Integer> integers = 
    CollectionUtils.collect(strings, s -> Integer.parseInt(s)); 
// [1, 2, 3]

  • ​（5）安全处理空集合

    List<String> list = null;

// 避免空指针异常
if (CollectionUtils.isEmpty(list)) {
    System.out.println("集合为空");
}

回忆：Java 中的集合框架（Java Collections Framework）​

  • ​根接口：Collection<E>，定义了集合的基本操作。
  • ​核心子接口：
接口 说明 常见实现类
​List 有序集合，允许重复元素 ArrayList, LinkedList
​Set 无序集合，元素唯一 HashSet, TreeSet
​Queue 队列，先进先出（FIFO）或优先队列 LinkedList, PriorityQueue
​Deque 双端队列，支持两端插入和删除 ArrayDeque
  • ​示例代码： 
    // List 示例：允许重复，按插入顺序存储
List<String> list = new ArrayList<>();
list.add("Apple");
list.add("Banana");
list.add("Apple");  // 允许重复

// Set 示例：元素唯一，无序
Set<Integer> set = new HashSet<>();
set.add(1);
set.add(2);
set.add(1);  // 重复元素会被忽略

// 遍历集合
for (String fruit : list) {
	System.out.println(fruit);
}

MyBatis

QueryWrapper

  • MyBatis-Plus（MP）​ 框架中用于动态构建 SQL 查询条件的核心工具类。它通过链式调用方法（如 eq(), like(), between() 等）生成 WHERE 子句，替代手写 SQL，提升开发效率和代码可读性。

使用步骤（Spring Boot + MyBatis-Plus）

  • ​步骤 1：添加 MyBatis-Plus 依赖，在 pom.xml 中引入依赖：

    <dependency>
    <groupId>com.baomidou</groupId>
    <artifactId>mybatis-plus-boot-starter</artifactId>
    <version>3.5.3.1</version> <!-- 使用最新版本 -->
</dependency>

  • 步骤 2：定义实体类
    假设有一个 User 表，对应实体类如下：

    @Data
@TableName("user") // 表名映射（若表名与类名一致可省略）
public class User {
    @TableId(type = IdType.AUTO) // 主键自增
    private Long id;
    private String username;
    private Integer age;
    private String email;
}

  • ​步骤 3：创建 Mapper 接口
    继承 BaseMapper，获得 MyBatis-Plus 的 CRUD 方法：

    public interface UserMapper extends BaseMapper<User> {
    // 可自定义 SQL 方法（此处不写，直接使用 BaseMapper 的方法）
}

  • ​步骤 4：在 Service 或 Controller 中使用 QueryWrapper
    通过 QueryWrapper 构建查询条件，并调用 selectList(), selectOne(), update(), delete() 等方法。

​核心方法及示例

​（1）基础查询

@Autowired
private UserMapper userMapper;

// 查询所有 age > 18 的用户
public List<User> getUsersByAge(Integer age) {
    QueryWrapper<User> queryWrapper = new QueryWrapper<>();
    queryWrapper.gt("age", 18); // age > 18
    return userMapper.selectList(queryWrapper);
}

// 查询 username = "张三" 的用户
public User getUserByName(String username) {
    QueryWrapper<User> queryWrapper = new QueryWrapper<>();
    queryWrapper.eq("username", username);
    return userMapper.selectOne(queryWrapper);
}

​（2）复杂条件组合

// 查询 age 在 20-30 之间，且 email 包含 "example.com" 的用户
public List<User> getUsersByCondition() {
    QueryWrapper<User> queryWrapper = new QueryWrapper<>();
    queryWrapper.between("age", 20, 30)
               .like("email", "example.com");
    return userMapper.selectList(queryWrapper);
}

// 查询 username 以 "张" 开头，或 age < 18 的用户
public List<User> getUsersByOr() {
    QueryWrapper<User> queryWrapper = new QueryWrapper<>();
    queryWrapper.likeRight("username", "张")
               .or()
               .lt("age", 18);
    return userMapper.selectList(queryWrapper);
}

​​（3）排序和分页

// 按 age 降序排序
public List<User> getUsersOrderByAge() {
    QueryWrapper<User> queryWrapper = new QueryWrapper<>();
    queryWrapper.orderByDesc("age");
    return userMapper.selectList(queryWrapper);
}

// 分页查询（需配置 MyBatis-Plus 分页插件）
public Page<User> getUsersByPage(Integer pageNum, Integer pageSize) {
    Page<User> page = new Page<>(pageNum, pageSize);
    QueryWrapper<User> queryWrapper = new QueryWrapper<>();
    return userMapper.selectPage(page, queryWrapper);
}

​​（4）更新和删除

// 将 age > 30 的用户 email 设为 null
public void updateUsersEmail() {
    User user = new User();
    user.setEmail(null);
    
    QueryWrapper<User> queryWrapper = new QueryWrapper<>();
    queryWrapper.gt("age", 30);
    userMapper.update(user, queryWrapper);
}

// 删除 username 包含 "test" 的用户
public void deleteUsersByName() {
    QueryWrapper<User> queryWrapper = new QueryWrapper<>();
    queryWrapper.like("username", "test");
    userMapper.delete(queryWrapper);
}

​高级用法

​（1）LambdaQueryWrapper（推荐）​
避免字段名硬编码，使用实体类属性的 getter 方法：

public User getUserByLambda(String username) {
    LambdaQueryWrapper<User> lambdaQuery = new LambdaQueryWrapper<>();
    lambdaQuery.eq(User::getUsername, username);
    return userMapper.selectOne(lambdaQuery);
}

（2）嵌套条件

// 查询 (age > 18 AND email 不为空) OR username = "admin"
public List<User> getUsersWithNested() {
    QueryWrapper<User> queryWrapper = new QueryWrapper<>();
    queryWrapper.and(wrapper -> wrapper.gt("age", 18).isNotNull("email"))
               .or()
               .eq("username", "admin");
    return userMapper.selectList(queryWrapper);
}

​​（3）联表查询
需配合自定义 SQL 和 @TableField 注解：

// 1. 实体类中添加关联字段（假设需要关联部门表）
public class User {
    // ...
    @TableField(exist = false) // 非数据库字段
    private String deptName;
}

// 2. Mapper 接口中定义方法
@Select("SELECT u.*, d.name AS dept_name FROM user u LEFT JOIN dept d ON u.dept_id = d.id ${ew.customSqlSegment}")
List<User> selectUserWithDept(@Param("ew") QueryWrapper<User> wrapper);

// 3. 使用 QueryWrapper 添加条件
public List<User> getUsersWithDept(String deptName) {
    QueryWrapper<User> queryWrapper = new QueryWrapper<>();
    queryWrapper.eq("d.name", deptName);
    return userMapper.selectUserWithDept(queryWrapper);
}
