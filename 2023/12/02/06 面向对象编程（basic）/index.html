06 面向对象编程（basic）

面向对象部分正式开始（教程的正式开始）
对应部分：P191 - P263
注意：比前面的内容重要一些，属于对OOP的回顾与重温

类与对象

  • 使用原因：
    • 单独用变量解决不利于数据的管理
    • 使用数组无法体现数据类型，并且获取信息时只能通过下标获得，造成变量名和数据的对应关系不明确
  • 类与对象的关系
    • 类：一个数据类型
      • 属性
      • 行为
    • 对象：类的一个具体实例
  • 对象在内存中的存在形式
    • 如果存放的是字符串，存放的是一个地址，指向方法区中的常量池里的内容
    • 如果是基本数据类型，会直接储存
    • ![[Snipaste_2023-12-02_15-54-04.jpg]]
  • 属性/成员变量
    • 属性是类的一个组成部分，一般是基本数据类型，也可以是引用数据类型
    • 注意事项
      • 属性的定义语法和变量一致
        • 访问修饰符 属性类型 属性名;
        • 访问修饰符：public protected 默认 privated
      • 属性的定义类型可以为任意类型，可以是基本数据类型，也可以是引用数据类型
      • 属性如果不赋值，默认赋值方式和数组一致
        • 创建后，如果没有赋值，有默认值
        1. int 0, short 0, byte 0, long 0
        2. float 0.0, double 0.0, char \u0000
        3. boolean false, String null
  • 创建对象的方式
    • 先声明再创建
      • Cat cat;
      • cat = new Cat();
    • 直接创建
      • Cat cat = new Cat();
  • 访问属性的方法
    • 对象名.属性
  • 类和对象的内存分配机制
    • 引用传递，是地址不是值
    • 创建对象流程：
      1. 先加载Person类信息(属性和方法信息，只会加载一次)
      2. 在堆中分配空间，进行默认初始化
      3. 把地址赋给p，p就指向对象
      4. 进行指定的初始化
  • 成员方法
  • 方法的定义与细节
    • 成员方法的好处
      • 提高了代码的复用性
      • 可以将实现的细节封装起来，供其他用户调用
    • 定义
      • 访问修饰符 返回数据类型 方法名(形参列表){语句 return返回值}
    • 注意事项和使用细节
      • 访问修饰符：
        • 控制方法的适用范围，如果不写则是默认访问
      • 返回类型：
        • 一个方法最多有一个返回值（返回多个结果：返回数组）
        • 返回类型可以为任何类型，包含基本类型或引用类型
        • 如果方法要求有返回数据类型，则方法中最后的执行语句必须为return语句，并且返回值的类型必须与return*的数值类型一致或兼容
        • 如果是void，可以没有return语句或只写return;
      • 方法名命名：
        • 驼峰命名法，需要表明函数的作用
      • 参数列表
        • 一个方法可以有0个参数，也可以有多个参数，中间用逗号隔开
        • 参数可以为任何类型，包括基本类型和引用类型
        • 调用带参数的方法时，一定要对应着参数列表传入相同类型或兼容类型
        • 形参和实参类型一致或兼容，个数相同，顺序相同
      • 方法体
        • 里面的语句可以为输入输出，变量，运算、分支、循环、方法调用等，但是里面不能再定义方法，即：方法不能嵌套定义
      • 方法调用
        • 同一个类中的方法可以相互调用
        • 跨类调用需要通过对象名调用
  • 成员方法传参机制
    • 基本数据类型的传递是值拷贝，形参的改变不影响实参
    • 引用数据类型的传递是地址传递，可以通过形参影响实参
  • 方法递归（recursion）调用
    • 递归的执行机制
  • 递归规则
    • 执行一个方法时，就会创建一个新的受保护的独立空间
    • 方法的局部变量时独立的，不会相互影响
    • 如果方法中使用的是引用类型变量（如数组，对象），会共享该引用类型的数据
    • 递归必须向退出递归的条件逼近，否则就是无限递归，出现栈溢出Stack Over Flow Error问题
    • 当一个方法执行完毕或遇到return，就会返回，谁调用的就把结果返回给谁

  • 迷宫问题
